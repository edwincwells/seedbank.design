# Documentation

The guides for working with a clone of seedbank.design. Read in this order if you're new; jump straight to one if you know what you need.

| Guide | What it's for | Who reads it |
|---|---|---|
| [`components.md`](components.md) | Getting a page up, composing components, choosing between near-neighbours. The build-a-page guide. | Anyone assembling something with the system |
| [`adapting.md`](adapting.md) | Rebranding and changing the system — the token axes, colour rules, and the editor reference tables for surfaces that need hand-maintenance. | Anyone editing `tokens.css` or writing new component CSS |
| [`testing-lowfi.md`](testing-lowfi.md) | Validating your changes — the three-criterion bar, and the checks that catch the failures that happen silently. | Anyone before they commit |

The canonical acceptance bar lives in [`../TESTING.md`](../TESTING.md); the design principles in [`../PRINCIPLES.md`](../PRINCIPLES.md); per-component detail in each component's own `README.md`. These guides are the connective tissue between those — they cross-reference rather than restate.
