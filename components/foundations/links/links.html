links

Inline link styling. Every <a href> in body prose, lists, tables, and component content uses this styling by default — no classes needed.

a[href] · default state

The seedbank.design project lives at github.com/edwincwells/seedbank.design as a template repository. Links inside body prose are underlined, coloured with --link, and inherit the surrounding text's size and weight.

a[href]:hover · thicker underline

On hover, the underline thickens from 1px to 2px. Colour does not change — the affordance is weight, not chroma. Hover over the link above to see the effect.

a[href]:visited · hue-shifted indigo

A visited link will render in --link-visited, a desaturated indigo that signals "you've been here" without pulling toward --make (system purple). Try this link to a well-known page: example.com. After visiting once and returning, the colour will have shifted.

Note: :visited styling is restricted by browsers for privacy. Only colour changes will be visible; thickness or other properties applied to :visited are ignored by design.

a[href]:focus-visible · focus ring from foundations/focus-ring

Tab to this link to see the focus ring. Focus styling is shared with every other focusable element in the system — links do not override it. The ring sits outside the text, so it works regardless of underline or colour.

a[rel~="external"] · trailing ↗ glyph

External links are marked explicitly: mutual aid on Wikipedia. The rel="external" attribute triggers the trailing arrow — auto-detection on protocol is not used.

The glyph also works when rel contains multiple values: seed banks on Wikipedia has rel="external noopener" and still renders the arrow.

a[href] inside other prose elements

Links inherit context where they can. Inside strong text, links keep the medium weight. Inside emphasis, links keep the italic. Inside smaller text, links keep the smaller size. Colour is the only thing the link asserts.

Links in headings

A link in a heading uses the heading's size and weight. The underline and colour still apply — there's no carve-out for headings.

a[href] on --surface-raised

Inside a raised card, links still use --link. The colour token was chosen to pass contrast on both --surface and --surface-raised, so no override is needed. Here is a link on a raised surface to verify.

<a> without href · not styled as a link

An anchor with no href is not a link and does not receive link styling. The selector scopes everything to a[href] so placeholders and target anchors stay neutral.

How forking works

Click Use this template at the top of the seedbank.design repository on GitHub. GitHub will create an independent repository in your account — no link back to upstream, no obligations, no approval process for your changes.

The token system is the foundation. Every visual property is a CSS custom property with a value-based name (see tokens.css in the repo). Remap them without touching component files.

If you need help

The adaptation guide in docs/ covers common patterns. For more on the philosophy, the solarpunk Wikipedia entry is a reasonable starting point, though the movement's aesthetic is not codified and seedbank.design is broader in scope.
