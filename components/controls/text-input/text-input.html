text-input

Text-shaped form inputs and <textarea>. This component styles the input element itself; composition with labels, help text, and inline errors is handled by the form-field pattern. An input on its own with a wrapping <label> is a complete, accessible control — that's how the examples here are written.

input types — every text-shaped input renders the same way

<textarea> — vertical resize only, 80px minimum height

disabled — uses the disabled attribute, not interactive

readonly — focusable and submitted, but not editable

aria-invalid="true" — applied after validation, never on pristine fields

inline with a button — heights match by design

no visible label — aria-label is required

when not to use a text-input

Avoid — placeholder as label

The placeholder is auxiliary text. When the user starts typing, it disappears — leaving the field unlabelled. Screen readers may or may not announce the placeholder, depending on browser. Always provide a real <label> (or aria-label if no visible label is possible). Placeholders are for example values and format hints only.

Avoid — no label at all

Every input needs an accessible name. Either wrap the input in a <label>, associate a separate <label for="..."> with the input's id, or provide an aria-label attribute. An unlabelled input is invisible to screen readers and unusable for many users.

Avoid — marking pristine fields invalid

Don't apply aria-invalid="true" to fields the user hasn't interacted with yet. Marking every empty required field as invalid on page load makes the entire form look broken before the user has done anything. Apply the attribute after a validation attempt — on blur, on submit, or after the user has typed and corrected.

Avoid — type="number" for things that aren't quantities

Use type="number" only for actual quantities (counts, measurements). For phone numbers, postcodes, credit card numbers, or anything where the value is a sequence of digits but isn't a number to be calculated with, use type="text" with inputmode="numeric" instead. type="number" applies number-specific behaviour (up/down spinners, scientific notation parsing, locale formatting) that breaks these use cases.
