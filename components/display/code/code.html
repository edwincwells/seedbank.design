code

Inline <code> and block <pre><code> treatments. Applied globally via element selectors — no class needed. No syntax highlighting (composition concern for forks).

inline — <code> in running prose

The system's primary action colour is set via the --action token, which by default resolves to --grow. To swap the primary to a different activity colour, override the token in your fork's tokens.css.

Component classes follow a variant-name pattern: btn-secondary, badge-warning, tr-repair. State is driven by attributes — disabled, aria-invalid, aria-busy — not by class names like btn-disabled.

avoid — don't use <code> for emphasis or styling, only for actual code

The element is semantic. Using <code> to make text look monospace when it isn't code is wrong — screen readers will announce the content as code, and forks that change the mono font won't see the change applied to non-code uses. If you want monospace text for stylistic reasons, write a class for it; don't reach for <code>.
